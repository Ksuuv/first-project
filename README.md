# Шпаргалка по консоли и Git

```
pwd (путь где находимся)
cd ~ (переход в дом директ)
cd .. (вернуться в папку выше)
cd ~/Desktop (переход на раб стол)
cd /c или cd C:\ (переход на диск С)
cd / (переход в корневую директорию)
ls (какие файлы в папке)
ls -a (какие файлы в папке, вкл скрытые)
cp имя файла название папки (скопировать файл в папку)
mv имя файла название папки (переместить файл в папку)
rm имя файла (удалить файл)
mkdir название папки (создание папки)
rmdir название папки (удаление папки)
rm -r название папки (удалить папки и все внутри)
через && можно писать неск команд в одну строку
Tab позволяет быстро подставлять названия

создать папку условно git
перейти в нее в консоли
git clone https://github.com/Ksuuv/test.git
создать/изменить файл в папке
git status
git add название файла (или git add . или git add --all)
git commit -m "коммент"
git push
git commit -a -m (-am) "коммент" (добавить все файлы)
cat название файла (показывает содержимое локально)
Если изменили на github
git fetch
git pull
touch название файла (создание файла)
git log (лог коммитов)
git show (инфо последий коммит)
git blame название файла (изменения от всех авторов)
echo "что хотим добавить в файл" >> название файла (добавит строку)
echo "что хотим добавить в файл" > название файла (перезапишет все)
git diff (какие изменения были в файле можно указать два коммита)
git diff <название_ветки1> <название_ветки2> (какие изменения между двух веток)
git diff <коммит1> <коммит2> (какие изменения между двумя коммитами)
git diff ветка~1 ветка (можно сравнить прошлый коммит ветки с текущим, число-это на сколько древний коммит)
git reset HEAD~1 (отменить последнее изменения)
git checkout . (откатить все последние изменения)
git branch название ветки (создание новой ветки)
git branch (показывает все ветки)
git branch -a (отображает все ветки локальные и удаленные)
git checkout название ветки (переключиться на другую ветку)
git push -u название ветки (перенести данные с новой ветки на гитхаб)
git checkout -  (вернуться в мастер ветку)
git checkout -b название ветки (создание ветки и переход в нее)
git merge название второй ветки (перенести вторую ветку в текущую)
git dranch -m название ветки  другое название ветки (переименовать ветку без файлов)
git push origin :название ветки (удалить ветку с гитхаба)
git branch -d название ветки (удалить ветку локально)
git fetch origin название ветки (вернуть ветку с гитхаб)
git commit --amend --no-edit (если нужно дополнить последний коммит)
git commit --amend -m "Новое сообщение" (изменить текст коммента)

Хэш - уникальный идентификатор коммита, если файл изменить, хэш измениться
Через git log можно понять как хэш, кто автор, дата коммита и коммент
Обозначение HEAD это последний коммит, HEAD находиться в .git, в самом HEAD ссылка на последний коммит (хэш)
Сообщение к коммиту максимум 72 символа и должно быть информативно

git restore --staged <file> (если добавил лишний файл через git add)
git restore --staged . (если нужно убрать все файлы папки из коммита)
git reset --hard <commit hash> (откатить коммит полностью к указанному)
git restore <file> (откатить изменения если еще не коммитил)

Для игнора файлов в .gitignore:
# коммент
*.jpeg (все файлы с расширением .jpeg)
file?.txt (игнор fileA.txt, file2.txt и тд)
file[0-2].txt (игнор файлов с указ символами в [])
/todo.txt (игнорировать todo.txt в корне репозитория)
build/ (игнорировать папку build)
docs/**/tmp (игнор всех папок между)
docs/*/tmp (игнор только одной папки между)
*.jpeg !doge.jpeg (игнор всех .jpeg кроме doge.jpeg)
git status --ignored (посмотреть что игнорируется)
```
